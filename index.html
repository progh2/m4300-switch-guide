M4300-24X24F 스위치로
자동할당(DHCP) 서버 만들기

예전 학교 백본 스위치를 다시 살려서, 전시장 기기들에게 IP 주소를 자동으로 나눠주는 서버로 만드는 방법입니다. 중학생도 그대로 따라 할 수 있도록 한 단계씩 설명합니다.

⏱ 예상 소요 30~60분 🔧 NETGEAR M4300 시리즈 🌐 스위치=게이트웨이(L3) 구조

📋 이 문서로 무엇을 하나요?

전시장에 노트북·키오스크·작품 기기들을 켜고 랜선만 꽂으면 알아서 인터넷에 연결되게 만드는 것이 목표입니다. 그러려면 누군가 각 기기에게 IP 주소를 자동으로 나눠줘야 하는데, 그 역할을 이 M4300 스위치가 맡습니다. 이렇게 IP를 자동으로 나눠주는 기능을 DHCP 서버라고 부릅니다.

🧭 먼저! 네트워크 구조 & 내 값 적기

우리가 만들 구조는 이렇습니다. M4300 스위치가 인터넷 입구(게이트웨이) 역할을 하면서, 동시에 전시 기기들에게 IP를 나눠줍니다.

   [ 인터넷 / 학교 외부망 ]
            │  (고정 IP 1개)
            │
   ┌────────┴─────────┐
   │   M4300 스위치    │   ← ① 외부망 고정IP 받음(업링크 포트)
   │  (라우팅 + DHCP)  │   ← ② IP를 자동으로 나눠줌
   └──┬──┬──┬──┬──┬───┘
      │  │  │  │  │      ← 전시 기기들이 꽂는 포트
    💻 📺 🖥 🎮 📱      ← 꽂으면 IP 자동 할당!
💡 핵심 개념
전시 기기들은 192.168.50.xxx 같은 내부 IP를 자동으로 받습니다. 그 기기들이 인터넷으로 나갈 때는 모두 스위치(192.168.50.1)를 거쳐서, 스위치가 받은 외부망 고정 IP를 통해 나갑니다.

✍️ 내 값 적기 (한 번만 입력하면 아래 모든 명령어에 자동으로 들어갑니다)

학교/시설 담당자에게 받은 외부망 정보를 넣으세요. 내부망 값은 그대로 써도 되고 바꿔도 됩니다.

🌐 외부망(인터넷) — 담당자에게 받은 값

예시값입니다. 받은 값으로 바꾸세요.
인터넷으로 나가는 출구 주소
외부망 랜선을 꽂을 포트

🏠 내부 전시망 — 그대로 써도 좋아요

보통 내부 IP의 마지막을 0으로
구글 DNS. 그대로 권장
⚠️ 아주 중요 — 담당자에게 꼭 확인하세요
이 구조에서 스위치는 라우팅만 하고 주소 변환(NAT)은 하지 않습니다. 그래서 인터넷이 되려면 둘 중 하나가 필요합니다.
  • (권장) 외부망 담당자가 우리 내부 대역(192.168.50.0/24)을 우리 고정 IP(203.0.113.50)로 보내주도록 경로(라우팅)를 등록해 주거나,
  • 외부망 입구에 NAT(공유)를 해주는 공유기/방화벽이 이미 있어서 우리 내부 IP를 대신 변환해 주면 됩니다.
둘 다 어렵다면, 전 단계 질문의 "상위 공유기가 게이트웨이" 구조가 훨씬 쉽습니다. 막히면 이 문서를 만든 사람에게 다시 물어보세요.

0 준비물 챙기기

  • M4300-24X24F 스위치 + 전원 케이블
  • 콘솔 케이블 — 스위치 'Console' 포트용. 모델에 따라 RJ-45(랜선처럼 생김) 또는 USB-C / 미니USB 케이블입니다. 스위치 앞/옆면의 Console 글씨를 찾으세요.
  • 노트북(윈도우) — 설정을 입력할 컴퓨터
  • USB-시리얼 변환기 — RJ-45 콘솔 케이블을 쓰는데 노트북에 9핀 시리얼 단자가 없을 때 필요 (대부분 필요)
  • 랜선 2개 이상 — 외부망 연결용 1개, 테스트용 1개

노트북에 터미널 프로그램 설치

콘솔로 스위치와 "대화"하려면 터미널 프로그램이 필요합니다. PuTTY 또는 Tera Term을 추천합니다(무료).

💡 팁
윈도우에서는 PuTTY가 가장 간단합니다. 인터넷에서 "PuTTY 다운로드"로 검색해 설치하세요.

1 스위치에 접속하기 (콘솔 연결)

스위치는 처음엔 화면도 마우스도 없습니다. 그래서 콘솔 케이블로 노트북과 직접 연결해서 글자(명령어)로 대화합니다.

① 케이블 연결

  1. 콘솔 케이블의 한쪽을 스위치 Console 포트에 꽂습니다.
  2. 반대쪽(USB)을 노트북에 꽂습니다.
  3. 스위치 전원을 켭니다.

② COM 포트 번호 확인 (윈도우)

  1. 윈도우키 + X → 장치 관리자 클릭
  2. 포트(COM & LPT) 항목을 펼칩니다.
  3. USB Serial Port (COM3) 처럼 적힌 번호를 기억하세요. (숫자는 컴퓨터마다 다름)
ℹ️ 안 보이면
USB-시리얼 변환기 드라이버가 없을 수 있습니다. 변환기 제조사(예: FTDI, Prolific) 드라이버를 설치하세요.

③ PuTTY로 접속

  1. PuTTY를 실행합니다.
  2. 연결 종류(Connection type)를 Serial 로 선택합니다.
  3. Serial line 칸에 위에서 확인한 COM3 입력 (본인 번호로).
  4. Speed 칸에 115200 입력.
  5. Open 클릭.
⚙️ 콘솔 통신 설정값 (외워두면 좋아요)
속도 115200 · 데이터 비트 8 · 패리티 None · 정지 비트 1 · 흐름 제어 None  (줄여서 115200 8-N-1)

④ 로그인

검은 창이 뜨면 Enter를 한 번 누르세요. 아이디를 물어보면 아래처럼 입력합니다.

User: admin
Password: (그냥 Enter — 처음엔 비밀번호가 없습니다)

접속되면 화면 끝이 (M4300-24X24F) > 처럼 바뀝니다. 성공입니다! 🎉

❓ 비밀번호를 모른다? (예전 학교 비번이 걸려 있음)
예전에 설정한 비밀번호가 남아 로그인이 안 되면, 바로 다음 2단계 공장 초기화로 넘어가세요. 초기화하면 비밀번호도 사라집니다.

2 공장 초기화 (예전 학교 설정 지우기)

예전 백본 서버 때 설정이 그대로 남아 있으면 충돌이 납니다. 깨끗하게 지우고 시작합니다. 상황에 따라 두 가지 방법이 있어요.

1단계에서 로그인에 성공했다면 이게 가장 쉽습니다. 명령어 한 줄이면 됩니다.

enable
clear config
→ "Are you sure...?" 물으면  y  입력 후 Enter

모든 설정이 공장 출고 상태로 돌아갑니다. 잠시 후 다시 > 가 나오면 완료입니다.

💡 참고
clear config는 설정만 지웁니다. 안전하게 하려면 이어서 스위치를 재부팅하세요: reload 입력 → y.

비밀번호를 몰라서 로그인이 아예 안 될 때는 부트 메뉴로 초기화합니다. (콘솔이 연결된 상태여야 합니다.)

  1. 콘솔 창(PuTTY)을 열어둔 채로 스위치 전원을 껐다 켭니다.
  2. 부팅 글자들이 올라오다가 화면에 Checking for application... 같은 문구가 나오는 순간, 키보드 Q 를 꾹 눌러 부팅을 멈춥니다.
  3. 메뉴(번호 목록)가 나오면 6 을 입력하고 Enter. (보통 "Restore to factory defaults" 항목)
  4. 확인을 물으면 y. 초기화 후 자동으로 다시 부팅됩니다.
ℹ️ 타이밍이 중요
문구가 지나가면 그냥 부팅돼 버립니다. 조금 일찍부터 Q를 연타하거나 꾹 누르고 있으면 잡기 쉬워요. 놓쳤으면 다시 전원을 껐다 켜서 시도하세요.
🚨 되돌릴 수 없음
초기화하면 예전 설정은 완전히 사라집니다. 이 스위치를 재사용하는 게 목적이니 괜찮지만, 혹시 살려야 할 설정이 있는지 한 번만 더 확인하세요.

3 관리자 비밀번호 & 관리 IP 설정

이제부터 본격 설정입니다. 먼저 관리자 비밀번호를 정하고, 나중에 웹 브라우저로도 관리할 수 있게 준비합니다.

① 설정 모드로 들어가기

아래 명령은 앞으로 계속 씁니다. 설정을 시작할 때 항상 입력한다고 생각하세요.

enable
configure

화면 끝이 (M4300-24X24F) (Config)# 로 바뀌면 설정 모드입니다.

② 관리자 비밀번호 정하기

username admin password 전시회비번2026 level 15
→ 노랑 글씨를 원하는 비밀번호로 바꾸세요(영문/숫자 섞어 권장)
📝 적어두기
여기서 정한 비밀번호를 꼭 메모하세요. 잊으면 다시 2단계 초기화부터 해야 합니다.

③ 웹 접속이 되도록 (내부 IP는 4·5단계에서 함께 설정)

이 스위치는 내부망 IP(192.168.50.1)가 곧 웹 관리 주소가 됩니다. 그 IP는 5단계에서 만들어집니다. 웹 서버 기능만 미리 켜둡니다.

ip http server
ip http secure-server
→ 일반(http)과 보안(https) 웹 접속을 모두 켭니다
🖱️ 웹 화면(GUI)으로 하고 싶다면
아직은 웹에 접속할 IP가 없어서 콘솔로 진행합니다. 5단계까지 마치면 노트북을 스위치에 랜선으로 꽂고 브라우저 주소창에 https://192.168.50.1 을 치면 웹 화면이 나옵니다. 이후 설정은 웹으로도 할 수 있어요(각 단계의 웹으로 하기 탭 참고).

4 외부망(인터넷) 고정 IP 연결

스위치가 인터넷으로 나갈 수 있도록 업링크 포트에 외부망 고정 IP를 설정합니다. 그리고 인터넷으로 가는 길(기본 경로)을 알려줍니다.

설정 모드(configure)에서 입력합니다. (3단계에 이어서 하면 됩니다.)

! 라우팅(길 안내) 기능 켜기 — 스위치가 인터넷 입구 역할을 하게 함
ip routing

! 업링크 포트를 '라우팅 포트'로 만들고 외부망 고정IP 부여
interface 1/0/1
routing
ip address 203.0.113.50 255.255.255.0
exit

! 인터넷으로 나가는 기본 경로(모든 길은 외부 게이트웨이로)
ip route 0.0.0.0 0.0.0.0 203.0.113.1

이제 업링크 포트(1/0/1)에 외부망 랜선을 꽂으세요.

웹 화면(5단계 이후 접속 가능)에서는 메뉴를 따라갑니다. NETGEAR M4300 웹 UI 기준입니다.

  1. 라우팅 켜기: 상단 메뉴 Routing → IP → Basic → Global Configuration 에서 Routing Mode = Enable 선택 후 Apply.
  2. 업링크 포트에 IP 주기: Routing → IP → Advanced → IP Interface Configuration 에서 업링크 포트(1/0/1) 선택 → IP 203.0.113.50, Subnet 255.255.255.0 입력 → Apply.
  3. 기본 경로 추가: Routing → Routing Table → Route Configuration 에서 Add → Network/Subnet 모두 0.0.0.0, Next Hop = 203.0.113.1 입력 → Apply.
💡 더 쉬운 방법
라우팅·IP 설정은 웹 메뉴가 다소 복잡합니다. 이 부분만큼은 명령어로 하기 탭이 더 빠르고 실수가 적습니다.

5 ⭐ DHCP 서버 켜기 (IP 자동 분배)

이 문서의 핵심입니다! 전시 기기들에게 IP를 자동으로 나눠주도록 설정합니다.

먼저: 내부망 IP(기기들의 게이트웨이) 만들기

전시 기기들이 꽂는 포트들이 모인 내부망(기본 VLAN 1)에 스위치 IP를 줍니다. 이 IP가 곧 기기들의 게이트웨이이자 웹 관리 주소가 됩니다.

역시 설정 모드(configure)에서 이어서 입력합니다.

(1) 내부망 IP 부여 (VLAN 1)

interface vlan 1
routing
ip address 192.168.50.1 255.255.255.0
exit

(2) 스위치 IP 등 일부 주소는 나눠주지 않도록 제외

! 시작 주소 앞쪽(스위치·예약용)과 끝쪽을 분배에서 제외
ip dhcp excluded-address 192.168.50.1 192.168.50.100
ip dhcp excluded-address 192.168.50.200 192.168.50.254

※ 위는 "시작~192.168.50.100 사이"와 "192.168.50.200~끝 사이"를 빼서, 실제로는 192.168.50.100~192.168.50.200만 나눠주는 효과입니다.

(3) DHCP 풀(나눠줄 IP 묶음) 만들기

ip dhcp pool exhibition
network 192.168.50.0 255.255.255.0
default-router 192.168.50.1
dns-server 8.8.8.8 8.8.4.4
lease 0 12 0
exit

※ lease 0 12 0 = 빌려주는 시간 0일 12시간 0분. 전시회용으로 적당합니다(원하면 1 0 0 = 1일).

(4) DHCP 서비스 켜기 (스위치 시작!)

service dhcp
exit

마지막 exit로 설정 모드를 빠져나옵니다. 이제 포트에 기기를 꽂으면 IP가 자동으로 나갑니다! 🎉

웹 화면(브라우저 https://192.168.50.1 접속)에서 진행합니다. NETGEAR M4300 메뉴 기준입니다.

(1) 내부망 IP 만들기

  1. Routing → VLAN → VLAN Routing 에서 VLAN ID 1 의 라우팅을 활성화(Create/Enable).
  2. Routing → IP → Advanced → IP Interface Configuration 에서 VLAN 1 선택 → IP 192.168.50.1, Mask 255.255.255.0 입력 → Apply.

(2) DHCP 서버 켜기

  1. System → Services → DHCP Server → DHCP Server Configuration 이동.
  2. Admin Mode = Enable 선택 → Apply.
  3. 같은 화면 아래 Excluded Addresses에서 나눠주지 않을 주소 범위(192.168.50.1 등)를 추가합니다.

(3) DHCP 풀 만들기

  1. System → Services → DHCP Server → DHCP Pool Configuration 이동.
  2. Pool Name에서 Create 선택 → 이름 exhibition 입력.
  3. Type of Binding = Dynamic 선택.
  4. Network Number = 192.168.50.0, Network Mask = 255.255.255.0.
  5. Default Router = 192.168.50.1, DNS Server = 8.8.8.8 8.8.4.4.
  6. Lease Time = 12 hours (원하는 값) → Apply.

🆘 문제 해결 (막혔을 때)

콘솔 창에 글자가 안 보여요 / 깨져 나와요
  • 속도가 115200 인지 확인하세요. (구형은 9600일 수도 있어요 — 둘 다 시도)
  • COM 포트 번호가 맞는지 장치 관리자에서 다시 확인하세요.
  • 케이블을 다시 꽂고, Enter를 몇 번 눌러 보세요.
기기를 꽂아도 IP를 못 받아요
  • service dhcp 를 켰는지 확인 (show ip dhcp pool).
  • 내부망 IP(192.168.50.1)를 VLAN 1에 줬는지 확인 (show ip interface brief).
  • 기기를 업링크 포트(1/0/1)가 아닌 다른 포트에 꽂았는지 확인. 업링크 포트는 외부망 전용입니다.
  • 기기의 랜 설정이 "자동으로 IP 받기" 인지 확인.
IP는 받는데 인터넷이 안 돼요
  • 업링크 포트에 외부망 랜선이 제대로 꽂혔는지 확인.
  • 기본 경로가 맞는지 확인: show ip route 에 0.0.0.0/0 → 203.0.113.1 가 있어야 함.
  • 가장 흔한 원인: 외부망에서 우리 내부 대역(192.168.50.0/24)으로 돌아오는 길이 없음. 위 네트워크 구조의 주의 박스를 다시 보고, 외부망 담당자에게 경로 등록 또는 NAT를 요청하세요.

📖 용어 사전 (모르는 단어가 나오면)

  • 스위치 — 여러 기기를 랜선으로 연결해 주는 네트워크 장비. M4300은 그중 똑똑한(관리형) 스위치.
  • DHCP — 기기에게 IP 주소를 자동으로 나눠주는 기능. 이 문서의 목표.
  • IP 주소 — 네트워크에서 기기의 "집 주소". 예: 192.168.50.1
  • 게이트웨이 — 다른 네트워크(인터넷)로 나가는 "출입문". 우리 기기들의 출입문은 스위치.
  • 서브넷 마스크 — 같은 동네(네트워크)의 범위를 정하는 값. 보통 255.255.255.0.
  • DNS — 사람이 읽는 주소(naver.com)를 IP로 바꿔주는 "인터넷 전화번호부".
  • 라우팅 — 서로 다른 네트워크 사이에서 길을 안내하는 것. 스위치를 인터넷 입구로 만드는 기능.
  • NAT — 여러 내부 IP를 하나의 외부 IP로 바꿔 인터넷에 내보내는 기능. (M4300은 이걸 못해서 외부망의 도움이 필요)
  • 콘솔 — 화면 없는 장비를 케이블로 직접 연결해 글자로 설정하는 통로.
  • CLI — 명령어(글자)로 설정하는 방식. GUI — 마우스로 화면 클릭해 설정하는 방식(웹 화면).
  • 업링크 포트 — 외부망(인터넷 방향)으로 연결되는 포트.
  • VLAN — 하나의 스위치를 여러 개의 가상 네트워크로 나누는 기능. 우리는 기본값(VLAN 1)만 사용.
  • Lease(임대 시간) — 나눠준 IP를 얼마 동안 빌려주는지의 시간.
