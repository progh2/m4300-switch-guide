M4300-24X24F 스위치를
연결용 허브로 사용하기

예전 학교 백본 스위치를 전시장에서 여러 기기를 연결하는 스위치(허브)로 다시 사용하는 방법입니다. 복잡한 서버 설정 없이, 깨끗이 초기화하고 케이블만 잘 꽂으면 됩니다. 중학생도 그대로 따라 할 수 있어요.

⏱ 예상 소요 20~40분 🔧 NETGEAR M4300 시리즈 🔌 단순 연결(L2 스위치) 구조

📋 이렇게 쓰기로 했어요 (구조 설명)

원래는 스위치가 직접 IP를 나눠주는 DHCP 서버로 만들려 했지만, M4300은 NAT(주소 변환) 기능이 없어서 인터넷 공유가 까다롭습니다. 그래서 더 간단하고 안정적인 방법으로 바꿨습니다.

💡 새 역할: "연결만 해주는 똑똑한 멀티탭"
M4300은 여러 기기를 묶어 연결만 해줍니다. IP 자동할당(DHCP)과 인터넷 연결(NAT)은 외부망의 공유기/라우터(또는 학교망)가 담당합니다. 전시 기기들은 그 외부망에서 IP를 자동으로 받습니다.

전체 과정 한눈에 보기

  1. 준비물 챙기기 — 콘솔 케이블과 노트북(초기화할 때만 필요)
  2. 스위치 접속하기 — 콘솔로 연결
  3. 공장 초기화 — 예전 학교 설정을 깨끗이 지우기 (가장 중요!)
  4. (선택) 관리용 IP 설정 — 나중에 상태를 점검하고 싶을 때만
  5. 케이블 연결 — 외부망(공유기) ↔ 스위치 ↔ 전시 기기
  6. 테스트 — 기기가 IP 받고 인터넷 되는지 확인

🖥️ M4300-24X24F는 어떤 장비인가요?

집에서 쓰는 인터넷 공유기(IP공유기)는 다들 익숙하죠. M4300은 그것과 "사촌"쯤 되지만, 훨씬 크고 한 가지 일을 아주 잘하는 전문가용 스위치입니다. 공유기를 떠올리며 비교하면 이해가 쉬워요.

🏠 집 IP공유기

혼자서 다 합니다 — 인터넷 받아오기, IP 나눠주기(DHCP), 주소 변환(NAT), 와이파이, 방화벽까지. 대신 유선 포트는 보통 4개, 속도는 1Gbps 정도. 한 집 살림용 만능 일꾼.

🏢 M4300 스위치

"연결과 전달"을 전문으로 합니다. 포트가 매우 많고(48개) 아주 빠릅니다(각 10Gbps). 대신 NAT·와이파이는 없음. 빌딩의 거대한 배전반 같은 존재.

핵심 기능

  • 완전 관리형(Fully Managed): 웹 화면(GUI)·명령어(CLI)·콘솔로 세밀하게 설정 가능.

🧠 L2 · L3 · NAT, 그림으로 쉽게

네트워크 장비 설명에 꼭 나오는 세 단어입니다. 어렵게 들리지만 "우편물 배달"에 비유하면 쉽습니다.

① L2 (스위치 / "같은 동네 안에서 전달")

L2는 같은 동네(같은 네트워크) 안에서 기기끼리 데이터를 주고받는 일입니다. 각 기기는 MAC 주소라는 고유 번호(=집집마다의 문패)를 가지고 있고, 스위치는 "누가 어느 포트에 있는지" 외워뒀다가 정확한 포트로만 전달합니다. M4300의 기본이자 이번 전시회에서 쓰는 기능입니다.

        같은 동네 (한 네트워크)
   ┌──────────────────────────────┐
   │   💻A ── [ L2 스위치 ] ── 💻B  │
   │                 │             │
   │                💻C            │
   └──────────────────────────────┘
   A가 C에게 보내면 → 스위치가 C 포트로만 콕 집어 전달
   (문패=MAC 주소를 보고 같은 동네 안에서 배달)

② L3 (라우팅 / "다른 동네로 길 안내")

L3는 서로 다른 동네(다른 네트워크) 사이를 오가는 일입니다. 이때는 IP 주소(=도시+동네+번지가 적힌 주소)를 보고 길을 안내(라우팅)합니다. M4300은 이 기능도 됩니다(L3 스위치). 다만 이번 전시회 구조에서는 사용하지 않습니다.

   동네 1 (192.168.0.x)        동네 2 (10.0.0.x)
   ┌───────────┐   라우팅    ┌───────────┐
   │  💻 ── [ L3 라우터/스위치 ] ── 💻  │
   └───────────┘            └───────────┘
   "다른 동네 주소(IP)네? 그쪽으로 가는 길은 이쪽!"

③ NAT (주소 변환 / "여러 명이 대표 주소 하나로 외출")

NAT는 여러 기기의 내부 주소를, 인터넷으로 나갈 때 하나의 대표 주소로 바꿔주는 일입니다. 인터넷(외부)에는 우리 집 안의 개별 주소가 안 보이고, 대표 주소(공인 IP) 하나만 보입니다. 집 공유기가 매일 해주는 일이 바로 이것이고, 인터넷 공유의 핵심입니다.

   집 안 (사설 IP)                 인터넷 (공인 IP)
   💻 192.168.0.11 ┐
   📱 192.168.0.12 ┼─[ NAT 공유기 ]─→  203.0.113.50  →  🌐
   📺 192.168.0.13 ┘    (대표 주소 1개로 변환)
   외부에서는 셋 다 "203.0.113.50"으로 보임
🚨 그래서 M4300으로는 인터넷 공유가 안 됩니다
M4300은 L2(연결)와 L3(라우팅)는 되지만 NAT는 못 합니다. 즉 "여러 기기를 대표 주소 하나로 인터넷에 내보내는" 일을 할 수 없습니다. 그래서 그 일은 집 공유기 같은 NAT 장비에게 맡기고, M4300은 잘하는 일(빠른 연결=L2)만 시키는 것입니다.
🧩 이번 전시회 구조로 연결하면
인터넷 → [공유기: NAT·DHCP 담당] → [M4300: L2 연결 담당] → 전시 기기
각자 잘하는 일만 맡아서, 가장 간단하고 안정적입니다. 자세한 구조는 아래 네트워크 구조에서 이어집니다.

🧭 네트워크 구조

우리가 만들 연결은 이렇게 단순합니다. 스위치는 중간에서 신호를 전달만 합니다.

   [ 인터넷 ]
       │
   [ 공유기 / 라우터 ]   ← 고정 IP + DHCP + NAT 담당 (IP를 나눠줌)
       │  (랜선 1개)
   ┌───┴──────────────┐
   │   M4300 스위치    │   ← 연결만! (설정 거의 없음)
   └─┬──┬──┬──┬──┬────┘
     │  │  │  │  │       ← 전시 기기들이 꽂는 포트
   💻 📺 🖥 🎮 📱       ← 공유기에서 IP 자동 수신
⚠️ 꼭 확인 — IP는 누가 나눠주나요?
이 구조에서 스위치는 IP를 나눠주지 않습니다. 전시 기기가 IP를 자동으로 받으려면, 위쪽 외부망(공유기·라우터·학교망)에 DHCP 기능이 켜져 있어야 합니다.
  • 학교 외부망이 이미 IP를 자동으로 준다면 → 그대로 OK, 추가 장비 불필요.
  • 외부망이 IP를 안 준다면(고정 IP 하나만 줌) → 그 앞에 인터넷 공유기 1대를 두고 그 공유기가 DHCP·NAT를 하게 하세요. (시중 가정용 공유기로 충분)

💡 터미널 프로그램
콘솔로 접속하려면 PuTTY(무료)를 추천합니다. "PuTTY 다운로드"로 검색해 설치하세요.

1 스위치에 접속하기 (콘솔 연결)

예전 학교 설정을 지우려면 스위치와 "대화"해야 합니다. 콘솔 케이블로 노트북과 직접 연결합니다.

① 케이블 연결 → ② COM 포트 확인

  1. 콘솔 케이블을 스위치 Console 포트와 노트북에 꽂고, 스위치 전원을 켭니다.
  2. 윈도우키+X → 장치 관리자 → 포트(COM & LPT) 에서 COM3 같은 번호를 확인합니다. (번호는 컴퓨터마다 다름)

③ PuTTY로 접속

  1. PuTTY 실행 → 연결 종류 Serial 선택
  2. Serial line에 COM3(본인 번호), Speed에 115200 입력 → Open
⚙️ 콘솔 통신 설정값
속도 115200 · 데이터 8 · 패리티 None · 정지 1 · 흐름제어 None (= 115200 8-N-1)

④ 로그인

검은 창에서 Enter를 누르고 아래처럼 입력합니다.

User: admin
Password: (그냥 Enter — 처음엔 비밀번호 없음)
❓ 예전 비밀번호가 걸려 있어 로그인이 안 되면
바로 다음 2단계 · 방법 B(부트메뉴 초기화)로 가세요. 초기화하면 비밀번호도 사라집니다.

2 공장 초기화 — 가장 중요!

예전 백본 서버 설정(옛 VLAN·IP·라우팅 등)이 남아 있으면 단순 연결이 제대로 안 될 수 있습니다. 깨끗이 지우면 모든 포트가 기본 상태로 돌아가 바로 허브처럼 동작합니다.

1단계에서 로그인에 성공했다면 명령어 한 줄이면 됩니다.

enable
clear config
→ "Are you sure...?" 물으면  y  입력 후 Enter

이어서 재부팅하면 더 확실합니다: reload 입력 → y.

비밀번호를 몰라 로그인이 안 될 때는 부트 메뉴로 초기화합니다.

  1. 콘솔 창을 열어둔 채 스위치 전원을 껐다 켭니다.
  2. 화면에 Checking for application... 가 나오는 순간 Q 를 꾹 눌러 부팅을 멈춥니다.
  3. 메뉴가 나오면 6 입력 → Enter (보통 "Restore to factory defaults").
  4. 확인 물으면 y. 자동으로 다시 부팅됩니다.
ℹ️ 타이밍을 놓쳤다면
그냥 부팅돼 버립니다. 전원을 다시 껐다 켜고 처음부터 Q를 연타하세요.
✅ 초기화만 하면 사실상 끝!
공장 초기화된 M4300은 모든 포트가 같은 그룹(기본 VLAN)에 묶여 있어, 전원만 켜고 케이블을 꽂으면 곧바로 일반 스위치(허브)처럼 동작합니다. 아래 3단계는 선택, 4단계는 케이블 연결입니다.

3 (선택) 관리용 IP 설정

꼭 필요하진 않습니다. 다만 나중에 웹 화면으로 스위치 상태(포트 연결, 트래픽 등)를 점검하고 싶다면 관리용 IP를 하나 정해 두면 편합니다. 점검이 필요 없으면 이 단계를 건너뛰어도 됩니다.

📌 관리 IP는 외부망과 같은 동네로
관리 IP는 외부망(공유기)이 쓰는 IP 대역과 같은 범위로 정해야 접속됩니다. 예) 공유기가 192.168.0.x를 쓰면 스위치 관리 IP는 192.168.0.250처럼 안 겹치는 번호로.

✍️ 관리용 값 적기 (아래 명령어에 자동 반영)

외부망 대역에서 안 겹치는 번호
외부망 공유기 주소
메모해 두세요!
enable
configure

! 관리자 비밀번호 정하기
username admin password 전시회비번2026 level 15

! 관리용 IP 설정 (기본 VLAN 1에 부여)
interface vlan 1
ip address 192.168.0.250 255.255.255.0
exit

! 점검 PC에서 다른 대역으로 접속할 때를 위한 기본 경로
ip default-gateway 192.168.0.1

! 웹 접속 켜기
ip http server
ip http secure-server
exit

! 저장 (필수!)
write memory

※ 펌웨어에 따라 interface vlan 1 대신 network parms 계열 명령으로 관리포트 IP를 줄 수도 있습니다. 위 방식이 안 되면 4·5단계 연결만으로도 충분히 동작하니 건너뛰세요.

초기화 직후 노트북을 스위치에 직접 꽂고, 노트북 IP를 잠시 169.254.x.x 또는 스위치 기본 대역으로 맞춘 뒤 브라우저로 접속해 설정하는 방법입니다. 펌웨어별로 다르므로, 명령어 방식이 더 간단합니다.

  1. 브라우저에서 스위치 접속 후 로그인(admin / 빈 비번).
  2. System → Management → Network Interface → IPv4 Network Configuration 에서 관리 IP 192.168.0.250, Mask 255.255.255.0, Gateway 192.168.0.1 입력 → Apply.
  3. 오른쪽 위 Save 로 저장.
💾 저장 잊지 마세요
설정 후 write memory(또는 웹 Save)를 안 하면 전원을 끌 때 사라집니다.

4 케이블 연결하기

가장 중요한 실전 단계입니다. 순서대로 꽂으면 됩니다.

  1. 공유기 ↔ 스위치: 외부망 공유기(또는 학교망)의 LAN 포트에서 나온 랜선을 스위치의 아무 포트에 꽂습니다. (M4300은 어느 포트든 업링크로 쓸 수 있습니다)
  2. 스위치 ↔ 전시 기기: 노트북·키오스크 등 전시 기기들을 스위치의 다른 포트들에 각각 꽂습니다.
  3. 모든 기기의 랜 설정은 "자동으로 IP 받기(DHCP)" 로 둡니다.
🚨 절대 하지 말 것 — 루프(고리) 만들기
랜선 양쪽을 같은 스위치의 두 포트에 꽂거나, 두 스위치를 케이블 2개로 동시에 연결하면 네트워크 전체가 멈출 수 있습니다(브로드캐스트 폭풍). 한 연결당 케이블은 1개만, 고리가 생기지 않게 하세요.
💡 10G SFP+ 포트
M4300-24X24F는 일반 RJ-45 10G 포트 24개와 SFP+ 광 포트 24개가 있습니다. 전시 기기는 보통 RJ-45 포트에 꽂으면 됩니다. SFP+는 광모듈/DAC 케이블이 있을 때만 사용하세요.

5 테스트하기

  1. 전시 기기(또는 노트북)를 스위치 포트에 꽂습니다.
  2. 잠시 후 기기가 외부망 공유기에서 IP를 자동으로 받으면 연결 성공!
  3. 브라우저로 아무 사이트나 열리면 인터넷까지 성공 🎉
윈도우에서 받은 IP 확인하는 법

윈도우키+R → cmd → 검은 창에 ipconfig 입력. "IPv4 주소"가 나오면(예: 192.168.0.x) 정상입니다. 169.254.x.x가 나오면 IP를 못 받은 것 → 문제 해결 참고.

관리 IP를 설정했다면(3단계), 점검 PC 브라우저에서 https://192.168.0.250 으로 들어가 포트 상태를 볼 수 있습니다.

🖱️ 웹 화면(GUI)으로 관리하기

명령어(콘솔)가 부담스럽다면, 집 공유기 설정창처럼 웹 브라우저 화면으로 스위치를 관리할 수 있습니다. 포트가 잘 연결됐는지, 속도는 얼마인지 등을 마우스 클릭으로 확인하는 방법입니다.

ℹ️ 먼저 필요한 것
웹 접속을 하려면 스위치에 관리용 IP가 있어야 합니다. 3단계(선택)에서 관리 IP를 설정했다면 바로 접속하면 되고, 안 했다면 아래 "방법 B"로 노트북을 직접 연결해 접속하세요.

① 웹 화면 접속하기

  1. 점검용 노트북을 스위치에 랜선으로 연결합니다.
  2. 노트북이 관리 IP와 같은 동네(대역)인지 확인합니다. (예: 둘 다 192.168.0.x)
  3. 브라우저 주소창에 https://192.168.0.250 입력 → 이동.
  4. "보안 경고"가 떠도 고급 → 계속 진행을 누릅니다. (스위치 자체 인증서라 안전)

관리 IP를 안 정했을 때, 노트북을 스위치에 직접 연결해 접속하는 방법입니다.

  1. 노트북 랜선을 스위치의 아무 포트에 직접 꽂습니다.
  2. 노트북의 IP를 수동(고정)으로 잠깐 바꿉니다. (제어판 → 네트워크 → 어댑터 설정 → IPv4 속성)
    예: IP 169.254.100.50, 마스크 255.255.0.0 (스위치 기본 대역에 맞춤)
  3. 스위치 콘솔에서 show network 또는 show ip interface brief 로 현재 스위치 IP를 확인합니다.
  4. 브라우저에 그 IP를 http://(확인한 IP) 형태로 입력해 접속합니다.
  5. 점검이 끝나면 노트북 IP를 다시 "자동으로 IP 받기"로 되돌립니다.
💡
직접 연결 접속이 까다로우면, 3단계에서 관리 IP를 먼저 설정해 두는 편이 훨씬 쉽습니다.

② 로그인

아이디 admin, 비밀번호는 3단계에서 정한 값(처음이면 비워둠)으로 로그인합니다.

💡 메뉴 이름이 조금 다를 수 있어요
펌웨어 버전에 따라 메뉴 위치(Basic/Advanced 구분 등)가 다를 수 있습니다. "Port", "IP", "VLAN", "Save" 같은 핵심 단어를 기준으로 찾으면 됩니다. 이번 전시회는 단순 연결이라, 사실 웹에서 만질 것은 거의 없고 포트 상태 확인 정도면 충분합니다.

🆘 문제 해결 (막혔을 때)

기기가 IP를 못 받아요 (169.254.x.x 가 나옴)
  • 가장 흔한 원인: 외부망(공유기)에 DHCP가 꺼져 있거나, 공유기가 스위치에 연결돼 있지 않음. 공유기↔스위치 랜선을 확인하세요.
  • 공장 초기화를 했는지 확인. 예전 VLAN 설정이 남아 포트가 분리돼 있을 수 있습니다 → 2단계 다시.
  • 기기 랜 설정이 "자동으로 IP 받기" 인지 확인.
  • 해당 포트의 LED가 켜지는지 확인(케이블/포트 불량 점검).
IP는 받는데 인터넷이 안 돼요
  • 이 구조에서 인터넷은 외부망 공유기가 책임집니다. 공유기 자체가 인터넷이 되는지 먼저 확인(공유기에 PC를 직접 꽂아 테스트).
  • 공유기의 게이트웨이·DNS 설정 점검.
콘솔 창에 글자가 안 보이거나 깨져요
  • 속도가 115200 인지 확인(구형은 9600일 수도).
  • COM 포트 번호 재확인, 케이블 재연결 후 Enter 몇 번.
갑자기 네트워크 전체가 느려지거나 멈췄어요

케이블 루프(고리)일 가능성이 높습니다. 최근에 꽂은 랜선을 빼고, 한 연결당 케이블 1개인지 확인하세요. → 4단계 경고

관리 IP로 웹 접속이 안 돼요
  • 점검 PC가 관리 IP와 같은 대역인지 확인(예: 둘 다 192.168.0.x).
  • write memory로 저장했는지 확인. 안 했으면 다시 설정 후 저장.
  • 관리 IP 설정이 까다로우면 굳이 안 써도 연결 자체엔 문제없습니다(선택 단계).

📖 용어 사전

  • 스위치 / 허브 — 여러 기기를 랜선으로 묶어 연결해 주는 장비. 이번엔 M4300을 이 용도로만 사용.
  • DHCP — 기기에게 IP 주소를 자동으로 나눠주는 기능. 이번 구조에선 외부망 공유기가 담당.
  • NAT — 여러 내부 IP를 하나의 외부 IP로 바꿔 인터넷에 내보내는 기능. M4300은 이걸 못해서 공유기에 맡김.
  • 게이트웨이 — 인터넷으로 나가는 출입문. 이번엔 외부망 공유기.
  • 업링크 — 외부망(공유기) 방향으로 올라가는 연결.
  • VLAN — 스위치를 가상으로 나누는 기능. 초기화하면 모든 포트가 기본 VLAN 1에 묶여 단순 허브처럼 동작.
  • 콘솔 / CLI — 케이블로 직접 연결해 글자(명령어)로 설정하는 방식. GUI는 웹 화면 클릭 방식.
  • 루프(고리) — 케이블이 동그랗게 연결돼 신호가 무한히 도는 상태. 네트워크를 마비시키므로 금지.
